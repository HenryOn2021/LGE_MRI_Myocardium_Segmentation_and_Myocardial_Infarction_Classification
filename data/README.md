Data Layout
Raw medical imaging data are not included in this repository.
Segmentation
The segmentation pipeline expects the dataset split JSON generated by:
```bash
python main.py --stage data_prep --root_dir /path/to/project
```
By default, segmentation stages look for:
```text
<root_dir>/repository/dataset_config_train_test.json
```
Classification CSV Format
Classification CSV files should contain:
a binary `label` column, where `0` = MI absent and `1` = MI present
one filename/case identifier column; the scripts auto-detect common names such as:
`filename`
`file`
`case`
`id`
`name`
`image_name`
Example:
```csv
filename,label
case_001.nii.gz,0
case_002.nii.gz,1
```
2D Classification
Expected image inputs:
```text
emidec_2d_images/*.nii.gz
imperial_2d_images/*.nii
```
Optional infarct masks may be supplied for CAM/Dice analysis.
3D Classification
Expected image inputs:
```text
emidec_3d_images/*.nii.gz
imperial_3d_images/*.nii
```
